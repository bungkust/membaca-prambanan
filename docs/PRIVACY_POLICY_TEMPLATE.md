# Privacy Policy Template - Reference

Dokumen referensi untuk memahami setiap section dalam Privacy Policy dan requirement Google Play Store.

## Privacy Policy Sections Explained

### 1. Pendahuluan (Introduction)
**Purpose**: Memberikan context tentang aplikasi dan komitmen terhadap privasi.

**Required Content:**
- App name
- Purpose statement
- Commitment to privacy protection
- Target audience mention (children)

**Example:**
```
Selamat datang di [App Name]! Kami menghargai privasi Anda dan 
berkomitmen untuk melindungi informasi pribadi anak-anak yang 
menggunakan aplikasi kami.
```

### 2. Informasi yang Dikumpulkan (Data Collection)
**Purpose**: Transparan tentang data apa yang dikumpulkan.

**Required Content:**
- ✅ Data yang dikumpulkan (jelas dan spesifik)
- ❌ Data yang TIDAK dikumpulkan (important untuk children's apps)
- How data is collected
- Where data is stored

**For Children's Apps:**
- Must clearly state: No personal information
- Must list what IS collected (even minimal)
- Must explain why (functionality)

**Example:**
```
Aplikasi menyimpan data berikut secara lokal di perangkat:
- Progress belajar (localStorage)
- Riwayat sesi kuis
- Pengaturan aplikasi

Data ini TIDAK dikirim ke server dan hanya tersimpan di perangkat Anda.

Kami TIDAK mengumpulkan:
- Nama, email, atau informasi identitas
- Lokasi GPS
- Foto atau video
- Data dari aplikasi lain
```

### 3. Izin yang Digunakan (Permissions)
**Purpose**: Explain setiap permission dan why it's needed.

**Required Content:**
- List semua permissions
- Purpose untuk setiap permission
- Justification (why required)
- What data is accessed (if any)

**For COPPA Compliance:**
- Must justify every permission
- Explain minimal permissions only
- No dangerous permissions untuk children's app

**Example:**
```
INTERNET: Digunakan untuk memuat konten aplikasi. Tidak mengirim data pribadi.

Audio (output): Digunakan untuk Text-to-Speech. TIDAK merekam audio.
```

### 4. Bagaimana Data Digunakan (Data Usage)
**Purpose**: Explain how collected data is used.

**Required Content:**
- Specific use cases
- Data tidak dibagikan ke third party
- Data tidak dijual
- Data usage limited to functionality

**Example:**
```
Data digunakan semata-mata untuk:
- Menyimpan progress belajar
- Menampilkan riwayat
- Menyediakan pengalaman personal

Kami TIDAK membagikan atau menjual data ke pihak ketiga.
```

### 5. COPPA Compliance (Untuk Children's Apps)
**Purpose**: Declare COPPA compliance dan explain measures.

**Required Content:**
- COPPA compliance statement
- List of COPPA measures taken
- No personal data collection
- No behavioral advertising
- Age-appropriate content

**Example:**
```
Aplikasi ini dirancang khusus untuk anak-anak berusia 3+ dan mematuhi COPPA:
- Tidak mengumpulkan personal information
- Tidak menggunakan behavioral advertising
- Data hanya stored locally
```

### 6. Keamanan Data (Data Security)
**Purpose**: Explain security measures.

**Required Content:**
- Encryption (in transit, at rest)
- Data storage location
- Access controls
- Security practices

**Example:**
```
- Semua komunikasi HTTPS (terenkripsi)
- Data disimpan lokal, tidak di server
- Tidak ada database server
- Tidak require login/account
```

### 7. Hak Pengguna (User Rights)
**Purpose**: Explain user rights (data deletion, access, etc).

**Required Content:**
- How to delete data
- How to revoke permissions
- How to contact for data requests
- User control options

**Example:**
```
Hak Anda:
- Menghapus data: Settings → Reset Progress
- Mencabut izin: Android/iOS settings
- Menghapus app: Uninstall akan hapus semua data lokal
```

### 8. Pembaruan Kebijakan (Policy Updates)
**Purpose**: Explain how policy updates work.

**Required Content:**
- How users will be notified
- Last updated date
- Change notification process

**Example:**
```
Kami dapat memperbarui kebijakan ini. Tanggal "Terakhir diperbarui" 
di bagian atas akan diperbarui. Kami menyarankan meninjau kebijakan 
ini secara berkala.
```

### 9. Kontak (Contact)
**Purpose**: Provide contact information for privacy questions.

**Required Content:**
- Email address
- Website (if any)
- Contact method for privacy concerns

**Important:**
- Must be real, accessible contact
- Must respond to privacy inquiries
- Email harus checked regularly

**Example:**
```
Jika ada pertanyaan tentang privasi:
Email: privacy@yourdomain.com
Website: https://yourdomain.com
```

### 10. Persetujuan (Consent)
**Purpose**: Legal statement about consent.

**Required Content:**
- Use = consent statement
- Withdrawal of consent option
- Legal basis

**Example:**
```
Dengan menggunakan aplikasi ini, Anda menyetujui kebijakan privasi ini.
Jika tidak setuju, mohon tidak menggunakan aplikasi ini.
```

## Google Play Store Requirements

### Must Include:
- ✅ Clear language (simple, not legal jargon)
- ✅ Comprehensive data collection disclosure
- ✅ Permissions explanation
- ✅ Data usage explanation
- ✅ Security measures
- ✅ User rights
- ✅ Contact information
- ✅ Last updated date

### For Children's Apps:
- ✅ COPPA compliance section
- ✅ Age-appropriate language
- ✅ Clear statement: No personal data
- ✅ No behavioral advertising statement
- ✅ Parent-friendly explanations

### Accessibility:
- ✅ Available in-app (Settings → Privacy Policy)
- ✅ Available via URL (for Play Store submission)
- ✅ Must be accessible via HTTPS
- ✅ Mobile-friendly formatting

## Template Structure

```
1. Introduction
   - App name
   - Commitment statement

2. Data Collection
   - What is collected
   - What is NOT collected
   - Where stored

3. Permissions
   - List all permissions
   - Purpose for each

4. Data Usage
   - How data is used
   - No sharing/selling

5. COPPA Compliance (if children's app)
   - Compliance measures
   - No personal data
   - No behavioral ads

6. Data Security
   - Encryption
   - Storage location
   - Access controls

7. User Rights
   - Data deletion
   - Permission revocation
   - Contact for requests

8. Policy Updates
   - Update process
   - Notification method

9. Contact
   - Email
   - Website
   - Response commitment

10. Consent
    - Use = consent
    - Withdrawal option
```

## Best Practices

1. **Clear and Simple Language**
   - Avoid legal jargon
   - Use plain Indonesian
   - Explain technical terms

2. **Be Specific**
   - Don't say "we collect data"
   - Say "we collect audio output for TTS functionality"
   - Be precise about what IS and ISN'T collected

3. **Regular Updates**
   - Update when app features change
   - Update last updated date
   - Notify users of significant changes

4. **Accessibility**
   - Make it easy to find (Settings menu)
   - Mobile-friendly format
   - Clear headings and sections

5. **Honesty**
   - Don't hide data collection
   - Be transparent about all practices
   - Don't make misleading statements

## Common Mistakes to Avoid

1. ❌ Generic privacy policy (copy-paste dari template lain)
2. ❌ Missing specific data collection details
3. ❌ Not explaining permissions clearly
4. ❌ Missing COPPA section for children's apps
5. ❌ Unclear or missing contact information
6. ❌ Outdated last updated date
7. ❌ Not accessible via HTTPS
8. ❌ Too complex language for parents to understand
